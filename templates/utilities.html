Load another paper file

Updates database_path in config.json and reloads the reader (same idea as the desktop “Load Another Paper File”).


Merge another paper file

Append all records from a second .cnt file. New numbers continue after the current maximum (desktop merge behavior).


Clean current database

Runs the same standardization as the desktop “Clean Current Database” (spacing, journal names, title case, etc.) and saves with the original file header preserved.


Save current as new paper file

Writes all in-memory records to a new path, reusing the header from the active file when possible (otherwise a default header).


Export full database

Download BibTeX (.bib) Download Excel (.xlsx)

Import BibTeX into current database

Parse a .bib file and either merge (new numbers after max) or replace all records (header preserved on replace).



Preferences (config.json)

Defaults used by the desktop app for new papers and reports. One vita type code per line (e.g. J, PA, B).
